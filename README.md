# ANOTE Web

Marketing website and live demo for **ANOTE** — an AI-powered Czech medical dictation assistant that transcribes doctor–patient conversations and generates structured medical reports.

**Live site**: [ashy-sea-04ea79a0f.1.azurestaticapps.net](https://ashy-sea-04ea79a0f.1.azurestaticapps.net)

## Tech Stack

- **Framework**: Next.js 16.2.2 (App Router, Turbopack)
- **Language**: TypeScript
- **Styling**: Tailwind CSS 4
- **Animations**: Framer Motion 12
- **Hosting**: Azure Static Web Apps
- **CI/CD**: GitHub Actions (auto-deploy on push to `main`)
- **Backend APIs**: Azure OpenAI (Whisper + GPT), ANOTE FastAPI backend

All pages support Czech (`/cs/...`) and English (`/en/...`). Czech is the default — visiting `/demo` resolves to `/cs/demo` via the proxy middleware.

## Live Demo

The `/demo` page provides a fully working demo where visitors can:

1. **Record voice** directly in the browser (up to 10 min, via MediaRecorder API)
2. **Upload audio files** (mp3, wav, m4a, webm — up to 25 MB, drag-and-drop supported)
3. **Watch live transcription** update every ~20 seconds during recording
4. **See a structured medical report** generated by GPT in real-time
5. **Select visit type** (auto-detect, initial exam, follow-up, gastroscopy, colonoscopy, ultrasound)
6. **Copy or download** the final report

The demo proxies through Next.js API routes — all API keys stay server-side. IP-based rate limiting prevents abuse.

## Getting Started

### Prerequisites

- Node.js 20+
- npm

### Install & Run

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

### Environment Variables

Create `.env.local` with:

```env
# Admin
ADMIN_SECRET=your-admin-password

# Contact form (Gmail SMTP)
CONTACT_EMAIL_TO=you@example.com
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=you@gmail.com
SMTP_PASS=your-app-password
SMTP_FROM=you@gmail.com

# Live demo — Azure Whisper API (transcription)
AZURE_WHISPER_ENDPOINT=https://<resource>.openai.azure.com/openai/deployments/whisper/audio/transcriptions?api-version=2024-06-01
AZURE_WHISPER_KEY=your-key

# Live demo — ANOTE backend (report generation)
ANOTE_BACKEND_URL=https://your-backend-url
ANOTE_API_TOKEN=your-token
```

All env vars are **server-side only** (no `NEXT_PUBLIC_` prefix).

## Deployment

### Azure Static Web Apps

The site deploys automatically via GitHub Actions on every push to `main`.

**Workflow**: `.github/workflows/azure-static-web-apps.yml`
- Installs dependencies (`npm ci`)
- Runs type checking (`tsc --noEmit`)
- Builds (`npm run build`)
- Deploys to Azure SWA

**Required GitHub secret**: `AZURE_STATIC_WEB_APPS_API_TOKEN`

**Azure app settings** (env vars) are configured via Azure Portal or CLI:
```bash
az staticwebapp appsettings set --name anote-web \
  --setting-names "AZURE_WHISPER_KEY=..." "ANOTE_API_TOKEN=..." ...
```

### Security Headers

Configured in `staticwebapp.config.json`:
- Content-Security-Policy (restricts connect-src to self + APIs)
- X-Frame-Options: DENY
- Referrer-Policy: strict-origin-when-cross-origin
- Permissions-Policy (allows microphone for demo)

## i18n

Two locales: Czech (`cs`, default) and English (`en`).

- Dictionaries: `src/dictionaries/cs.json`, `en.json`
- Routing: `[lang]` dynamic segment, proxy middleware handles locale detection
- EN slugs are mapped to CS filesystem routes (e.g. `/en/pricing` → `/en/cenik`)
